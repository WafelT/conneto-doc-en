Code Writing Patterns

General Code Writing Guidelines

  • Variables for functions, methods, and data should be declared in camelCase format.
  • All variable names should be written in English without grammatical faults and should clearly reflect the meaning of the entity.
  • Indentation: 4 spaces.
  • BEM methodology should be used in developing the markup.
  • Elastic layout is used; all values in px should be converted to vw (handled by the ‎ px2vw ‎ SCSS function) with some exceptions.

Working with Vue Components

Components are imported using arrow functions in the components key object. Global UI components from the /ui directory do not need to be imported; they are imported by default.

Advantage: This notation is compact and meaningful, allowing you to avoid unnecessary code clutter in the component file.

Recommendation: Try to separate imports of components with an empty line, grouping them for better readability.

Vuex | Working with Store

Created new store file should following the schema steps getters, mutations, actions, should be used as depicted in the schema. All requests to the Backend API should be defined only in the action functions of the store.

Advantage: Proper pattern for building architecture is followed, adhering to the single responsibility principle. Components remain clean and are easily maintainable.

Working with SVG

Adding new icons to the project is done exclusively through the sprite.svg file
(except for components like icons, whose styles are not applied through the sprite).

Advantage: Using SVG sprites provides a more flexible and efficient management of graphical elements, facilitating their development, maintenance, and optimization.

Working with Language Files

Changes to language files are carried out in batches, and language files are supplemented or modified strictly based on tasks related to updating static files.

The first step involves identifying the new additions or changes in the en.js file. Afterward, they are converted into a .xml file and sent to the client. The client returns a .xml file, which is then converted back into .js and added to the end of the file with the corresponding language. Additionally, the version in the file is updated.
